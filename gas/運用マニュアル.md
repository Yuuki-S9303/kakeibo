# 家計・資産管理システム 運用マニュアル

---

## 1. あなたがやること・システムが自動でやること

### 🙋 あなたがやること

| タイミング | アクション | 方法 |
|-----------|-----------|------|
| 現金で買い物した後 | レシートを撮影してLINEに送る | LINE画像送信 |
| 撮影できない場合 | 金額とメモをLINEに入力 | LINEテキスト |
| 財布の現金が変わった時 | 財布の残高をLINEに入力 | LINEテキスト |
| 毎月末 | 給与・銀行・証券の残高を入力 | LINEテキスト |

### 🤖 システムが自動でやること

| タイミング | 内容 |
|-----------|------|
| 10分ごと | 三井住友カード・PayPayのGmailを監視してスプシに自動記録 |
| 毎月1日 | 先月の収支レポートをLINEに送信 |
| 毎月25日 | 給与・資産の入力リマインダーをLINEに送信 |

## 2. LINEコマンド一覧

### 💰 現金残高の記録
```
残高 8000
残高 ¥8,000
```
→ cash_balanceに記録。前回との差分が自動でtransactionsに「現金支出」として登録される。

### 🛍️ 支出の手入力
```
1500 ランチ
800 コンビニ
```
→ transactionsに支出として登録。カテゴリは未設定（後でLIFFから設定可）。

### 📈 収入の手入力
```
収入 250000 給与（夫）
収入 180000 給与（妻）
収入 50000 副業
```
→ transactionsに収入として登録。

### 🏦 資産の記録
```
資産 みずほ 580000
資産 SBI証券 1250000
資産 持株会 320000
```
→ assetsに記録。

### 🧾 レシートOCR
LINEにレシート画像を送るだけ。
→ Gemini AIが店名・金額・カテゴリを自動抽出してtransactionsに登録。

## 3. データフロー

```
【入力元】                  【GAS処理】              【スプシ】

三井住友カードメール ──→ GAS（10分ごと自動実行） ──→ transactions
PayPayメール ────────→ GAS（10分ごと自動実行） ──→ transactions
                                                  mail_log（重複防止）

LINEレシート画像 ────→ Gemini OCR ──────────────→ transactions

LINEテキスト「残高」 ──→ GAS ──────────────────→ cash_balance
                                                  transactions（差分を現金支出として）

LINEテキスト「支出」 ──→ GAS ──────────────────→ transactions

LINEテキスト「収入」 ──→ GAS ──────────────────→ transactions

LINEテキスト「資産」 ──→ GAS ──────────────────→ assets
```

## 4. 各シートの役割

| シート | 役割 | 更新タイミング |
|--------|------|--------------|
| **transactions** | 全収支の明細（メイン） | 随時（自動・手動） |
| **cash_balance** | 財布の現金残高の履歴 | 残高変更のたびに手入力 |
| **assets** | 銀行・証券などの資産残高（月次スナップショット） | 月末に手入力 |
| **budgets** | カテゴリ別の月次予算 | 初期設定時・変更時のみ |
| **categories** | カテゴリマスタ | 初期設定時・変更時のみ |
| **summary** | 月次集計（自動生成） | GASが自動更新 |
| **mail_log** | 処理済みメールID（重複防止） | GASが自動更新 |

## 5. 月次の運用フロー

```
毎日
  ├─ カード・PayPay → 自動取得（何もしなくてOK）
  ├─ 現金払い → レシート撮影 or LINEで手入力
  └─ ATMで現金を引き出した時 → 「残高 ○○○○」を入力

毎月末（25日ごろ）
  ├─ LINEからリマインダーが届く
  ├─ 給与を入力：「収入 250000 給与（夫）」
  ├─ 銀行残高を入力：「資産 みずほ 580000」
  └─ 証券残高を入力：「資産 SBI証券 1250000」

毎月1日
  └─ LINEに先月の収支レポートが届く（自動）
```

## 6. カード・PayPayの自動取得の仕組み

GASが10分ごとにGmailを監視しています。

**三井住友カード**「ご利用のお知らせ」メールから：
- 日付・金額・店舗名を自動抽出
- transactionsに「三井住友カード」として登録

**PayPay**「決済完了」メールから：
- 日付・金額・店舗名を自動抽出
- transactionsに「PayPay」として登録

**重複防止：** 処理済みメールのIDをmail_logに記録するため、同じメールが二重登録されることはありません。

**メール形式が変わった場合：** 抽出に失敗するとLINEにアラートが届きます。その場合は手動で入力してください。

---

## 7. 現金残高の仕組み（重要）

財布の残高を入力するだけで現金支出が自動計算されます。

**例：**
```
前回記録：残高 10000（昨日）
今日入力：残高 7500

→ 差分 -2500円が「現金支出」としてtransactionsに自動登録
→ cash_balanceに7500円を記録
```

⚠️ **5000円以上減った場合**はLINEで確認メッセージが届きます。

**ATMで現金を引き出した場合：** 引き出し後に「残高 ○○○○」を入力してください。差分がプラスになるため支出は登録されません。銀行残高は月末にまとめて記録する運用のため、引き出し時点での銀行残高は更新しなくてOKです。

---

## 8. こんな時どうする？

| 状況 | 対応 |
|------|------|
| レシートを撮り忘れた | LINEで「金額 メモ」を手入力 |
| OCRの結果が違う | LIFFアプリから修正（Phase 4で対応予定） |
| カードのメールが来ない | spam/プロモーションフォルダを確認。GASのアカウントと受信アカウントが同じか確認 |
| 二重登録された | LIFFアプリから削除（Phase 4で対応予定） |
| LINEのBotが返答しない | Cloudflare Worker → GASの接続を確認 |
